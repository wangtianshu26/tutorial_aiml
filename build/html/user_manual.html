15. AIML与技术文档（用户手册）¶

15.1. DITA¶

DITA是达尔文信息分类体系架构的英文缩写，其英文全称为Darwin Information Typing Architecture，它使用XML语言来描述发布信息，是一种实现结构化写作的有效工具。DITA标准中包含的内容十分广泛，基本包含了面向数字出版物的各项元素，涵盖了从信息组织、编写、生成以及交付的整个流程，并且还提供了让使用者根据需求扩展和定制内容的机制。

在DITA中有四种种基本主题类型：任务主题（task topic）,概念主题（concept topic）,参引主题（reference topic），以及故障解决为主题（troubleshooting topic）。对于用户而言，使用DITA组织内容结构是能够清楚地回答用户是什么、怎么做等问题，帮助用户在繁杂的内容中根据自己的目的找到相关信息。

15.2. 聊天机器人与用户文档¶

在聊天机器人与技术文档结合这方面，这里主要讨论技术文档中的“用户手册”，因为技术文档还包括许多其他类型的文档，比如开发文档、管理文档等等。由于DITA也是一种基于XML的信息结构，因此在这里将基于AIML的聊天机器人与用户文档相关联。但是笔者并非第一个提出这个想法的人。陈立[1]在她的论文中设计了一种能够实现技术文档与智能问答内容同步开发的工具，首先提出的将二者联系起来。她的论文主要从各个方面做了产品的设计和实验，并没有细节的涉及到AIML语言应该如何和基于DITA的用户手册的内容相对应。因此在下一节，笔者将会综合前面的AIML的语法，分享如何将用户手册的内容与AIML语言中的多种标签进行对应。

15.2.1. 现有的用户手册的形式及优缺点¶

作为技术文档的一个重要的部分，用户文档在产品的运营中扮演着重要的角色，用户在买到新产品时，免不了要看用户手册。现有的用户手册大部分会和产品一起送达用户，而现实中人们对于用户手册诟病很深，加上现在快节奏的生活和用户心理，对于大量文字的用户手册，使用者会有抵触心理，大多数情况下会联系客服要求人工服务，另外，纸质的用户手册如果用户保管不当，对后续产品使用也造成了一定困难。

15.2.2. 聊天机器人的优缺点¶

从这点来看，高效便捷的用户手册形式对于用户和产品生产者都至关重要。而这个教程所介绍的基于AIML的聊天机器人则可以在一定程度上解决上述问题。

  • 聊天机器人可以部署在网页页面或者小程序中，根据用户偏好提供视频或者文字类型的用户手册，减轻人工客服的压力

  • 聊天机器人可以帮助产品生产者不断改善产品的用户体验。通过统计用户访问最多的问题，产品生产者可以快速定位产品用户体验较差的部分，然后加以改进。

  • 通过与用户互动，聊天机器人渐进式的为用户提供信息，缓解用户对长篇文字的抵触心理。

当然，聊天机器人也存在一定缺点，比如它的智能程度有限，相比于人工客服，其智能程度还较低，因此很多用户更偏向人工客服解决问题。

在这一节中我们介绍了基于DITA的用户手册以及现有的用户手册的优缺点和聊天机器人的优缺点，在下一节，笔者将分享，如何将用户手册中的内容与AIML语言中的标签相关联。
